## **1. Lambda Functions & Functional Programming**

### Lambda Functions
- `lambda` creates small, one-line, anonymous functions
- Perfect for functional programming patterns
- Often used with `map()`, `filter()`, `sorted()`, etc.

**➤ Basic Lambda Examples:**


```python
# Basic lambda function
square = lambda x: x ** 2
print("Square of 5:", square(5))

# Lambda with multiple arguments
add = lambda x, y: x + y
print("Add 3 + 7:", add(3, 7))

# Lambda with conditional logic
max_of_two = lambda a, b: a if a > b else b
print("Max of 10, 15:", max_of_two(10, 15))

# Using lambda with built-in functions
numbers = [1, 2, 3, 4, 5]
squared_numbers = list(map(lambda x: x**2, numbers))
even_numbers = list(filter(lambda x: x % 2 == 0, numbers))

print("Original:", numbers)
print("Squared:", squared_numbers)
print("Even numbers:", even_numbers)

# Lambda for sorting complex data
students = [('Alice', 85), ('Bob', 92), ('Charlie', 78)]
students_by_grade = sorted(students, key=lambda student: student[1])
print("Students by grade:", students_by_grade)
```

## **2. Generators & Iterators**

### Generators
- Use `yield` to produce values lazily (one at a time)
- Memory efficient - generates values on demand
- State is maintained between calls
- Can be infinite sequences

### Iterators
- Objects implementing `__iter__()` and `__next__()` methods
- Foundation of Python's iteration protocol

**➤ Basic Generator Example:**


```python
# Basic generator function
def count_up_to(n):
    count = 1
    while count <= n:
        yield count
        count += 1

# Generator object (lazy evaluation)
gen = count_up_to(5)
print("Generator object:", gen)
print("Generator values:", list(count_up_to(5)))

# Infinite generator
def fibonacci():
    a, b = 0, 1
    while True:
        yield a
        a, b = b, a + b

# Get first 10 fibonacci numbers
fib_gen = fibonacci()
fib_numbers = [next(fib_gen) for _ in range(10)]
print("First 10 Fibonacci numbers:", fib_numbers)

# Generator expression (like list comprehension but lazy)
squares_gen = (x**2 for x in range(5))
print("Squares generator:", squares_gen)
print("Squares values:", list(squares_gen))

# Memory comparison
import sys
list_comp = [x**2 for x in range(1000)]  # List comprehension
gen_expr = (x**2 for x in range(1000))   # Generator expression

print(f"List comprehension memory: {sys.getsizeof(list_comp)} bytes")
print(f"Generator expression memory: {sys.getsizeof(gen_expr)} bytes")
```

**➤ Iterator Example:**


```python
# Basic iterator usage
nums = [1, 2, 3]
it = iter(nums)

print("Using next():")
print(next(it))  # 1
print(next(it))  # 2
print(next(it))  # 3

# Custom iterator class
class CountDown:
    def __init__(self, start):
        self.start = start
    
    def __iter__(self):
        return self
    
    def __next__(self):
        if self.start <= 0:
            raise StopIteration
        self.start -= 1
        return self.start + 1

# Using custom iterator
print("\nCountdown from 5:")
for num in CountDown(5):
    print(num)

# Iterator with range
print("\nRange iterator:")
range_iter = iter(range(3))
for num in range_iter:
    print(f"Range value: {num}")

# Iterator protocol in action
class EvenNumbers:
    def __init__(self, max_num):
        self.max_num = max_num
        self.current = 0
    
    def __iter__(self):
        return self
    
    def __next__(self):
        if self.current >= self.max_num:
            raise StopIteration
        else:
            self.current += 2
            return self.current - 2

print("\nEven numbers up to 10:")
for even in EvenNumbers(10):
    print(even)
```

## **3. Decorators**

### What are Decorators?
- Functions that modify or enhance other functions
- Implement the **decorator pattern**
- Use `@decorator_name` syntax
- Can be stacked multiple times
- Essential for frameworks like Flask, Django

**➤ Basic Decorator Example:**


```python
# Basic decorator
def my_decorator(func):
    def wrapper():
        print("Something is happening before the function is called.")
        func()
        print("Something is happening after the function is called.")
    return wrapper

@my_decorator
def say_hello():
    print("Hello!")

say_hello()

# Decorator with arguments
def timing_decorator(func):
    import time
    def wrapper(*args, **kwargs):
        start_time = time.time()
        result = func(*args, **kwargs)
        end_time = time.time()
        print(f"{func.__name__} took {end_time - start_time:.4f} seconds")
        return result
    return wrapper

@timing_decorator
def slow_function():
    import time
    time.sleep(1)
    return "Done!"

result = slow_function()
print("Result:", result)

# Decorator factory (decorator with parameters)
def repeat(times):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for _ in range(times):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(3)
def greet(name):
    print(f"Hello, {name}!")

greet("Alice")

# Class-based decorator
class CountCalls:
    def __init__(self, func):
        self.func = func
        self.count = 0
    
    def __call__(self, *args, **kwargs):
        self.count += 1
        print(f"Call {self.count} of {self.func.__name__}")
        return self.func(*args, **kwargs)

@CountCalls
def add_numbers(a, b):
    return a + b

print(add_numbers(2, 3))
print(add_numbers(5, 7))
print(add_numbers(1, 9))
```

## **4. Context Managers & Resource Management**

### What are Context Managers?
- Objects that define runtime context for executing code blocks
- Implement `__enter__()` and `__exit__()` methods
- Used with `with` statement for resource management
- Ensure proper cleanup (files, connections, locks)

**➤ Context Manager Examples:**


```python
# Built-in context manager (file handling)
print("=== File Context Manager ===")
with open("temp_file.txt", "w") as file:
    file.write("Hello, World!")
    print("File is open and being written to")
print("File is automatically closed")

# Reading the file
with open("temp_file.txt", "r") as file:
    content = file.read()
    print(f"File content: {content}")

# Custom context manager class
class DatabaseConnection:
    def __init__(self, db_name):
        self.db_name = db_name
        self.connection = None
    
    def __enter__(self):
        print(f"Connecting to database: {self.db_name}")
        self.connection = f"Connection to {self.db_name}"
        return self.connection
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        print(f"Closing database connection: {self.db_name}")
        if exc_type:
            print(f"Exception occurred: {exc_type.__name__}: {exc_val}")
        return False  # Don't suppress exceptions

# Using custom context manager
print("\n=== Custom Context Manager ===")
with DatabaseConnection("users_db") as db:
    print(f"Working with: {db}")
    print("Performing database operations...")

# Context manager using contextlib
from contextlib import contextmanager

@contextmanager
def timer_context(name):
    import time
    print(f"Starting {name}")
    start_time = time.time()
    try:
        yield
    finally:
        end_time = time.time()
        print(f"{name} completed in {end_time - start_time:.4f} seconds")

# Using contextmanager decorator
print("\n=== Timer Context Manager ===")
with timer_context("Data Processing"):
    import time
    time.sleep(0.5)
    print("Processing data...")

# Multiple context managers
print("\n=== Multiple Context Managers ===")
with open("file1.txt", "w") as f1, open("file2.txt", "w") as f2:
    f1.write("Content 1")
    f2.write("Content 2")
    print("Both files written")

# Cleanup files
import os
for filename in ["temp_file.txt", "file1.txt", "file2.txt"]:
    if os.path.exists(filename):
        os.remove(filename)
        print(f"Cleaned up {filename}")
```

## **5. Magic Methods (Dunder Methods)**

### What are Magic Methods?
- Special methods with double underscores (`__method__`)
- Called automatically by Python in specific situations
- Enable operator overloading and custom behavior
- Make objects behave like built-in types

**➤ Common Magic Methods:**
- `__init__`: Constructor
- `__str__`: String representation for users
- `__repr__`: String representation for developers
- `__len__`: Length of object
- `__getitem__`: Index access `obj[key]`
- `__add__`: Addition operator `+`
- `__eq__`: Equality comparison `==`

**➤ Magic Methods Example:**


```python
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __str__(self):
        """String representation for users"""
        return f"Vector({self.x}, {self.y})"
    
    def __repr__(self):
        """String representation for developers"""
        return f"Vector(x={self.x}, y={self.y})"
    
    def __add__(self, other):
        """Addition operator (+)"""
        return Vector(self.x + other.x, self.y + other.y)
    
    def __sub__(self, other):
        """Subtraction operator (-)"""
        return Vector(self.x - other.x, self.y - other.y)
    
    def __mul__(self, scalar):
        """Multiplication with scalar"""
        return Vector(self.x * scalar, self.y * scalar)
    
    def __eq__(self, other):
        """Equality comparison (==)"""
        return self.x == other.x and self.y == other.y
    
    def __len__(self):
        """Length/magnitude of vector"""
        return int((self.x**2 + self.y**2)**0.5)
    
    def __getitem__(self, index):
        """Index access"""
        if index == 0:
            return self.x
        elif index == 1:
            return self.y
        else:
            raise IndexError("Vector index out of range")

# Using magic methods
v1 = Vector(2, 3)
v2 = Vector(1, 4)

print("=== Magic Methods Demo ===")
print(f"v1: {v1}")          # Calls __str__
print(f"v2: {repr(v2)}")    # Calls __repr__
print(f"v1 + v2: {v1 + v2}") # Calls __add__
print(f"v1 - v2: {v1 - v2}") # Calls __sub__
print(f"v1 * 3: {v1 * 3}")   # Calls __mul__
print(f"v1 == v2: {v1 == v2}") # Calls __eq__
print(f"Length of v1: {len(v1)}") # Calls __len__
print(f"v1[0]: {v1[0]}, v1[1]: {v1[1]}") # Calls __getitem__

# Custom container class
class Playlist:
    def __init__(self, name):
        self.name = name
        self.songs = []
    
    def __str__(self):
        return f"Playlist: {self.name} ({len(self.songs)} songs)"
    
    def __len__(self):
        return len(self.songs)
    
    def __getitem__(self, index):
        return self.songs[index]
    
    def __setitem__(self, index, value):
        self.songs[index] = value
    
    def __contains__(self, song):
        return song in self.songs
    
    def __iter__(self):
        return iter(self.songs)
    
    def add_song(self, song):
        self.songs.append(song)

# Using custom container
print("\n=== Custom Container Demo ===")
playlist = Playlist("My Favorites")
playlist.add_song("Song 1")
playlist.add_song("Song 2")
playlist.add_song("Song 3")

print(playlist)                    # __str__
print(f"Length: {len(playlist)}")  # __len__
print(f"First song: {playlist[0]}") # __getitem__
print(f"'Song 1' in playlist: {'Song 1' in playlist}") # __contains__

# Iterate through playlist
print("All songs:")
for song in playlist:              # __iter__
    print(f"  - {song}")
```

## **6. Type Hints & Static Typing**

### What are Type Hints?
- Optional static typing annotations (Python 3.5+)
- Improve code readability and IDE support
- Enable static type checking with tools like `mypy`
- Don't affect runtime behavior
- Support generic types, unions, and custom types

**➤ Type Hints Examples:**


```python
from typing import List, Dict, Optional, Union, Tuple, Callable, Generic, TypeVar

# Basic type hints
def greet(name: str) -> str:
    return f"Hello, {name}!"

def add_numbers(a: int, b: int) -> int:
    return a + b

def calculate_average(numbers: List[float]) -> float:
    return sum(numbers) / len(numbers)

# Using the functions
print("=== Basic Type Hints ===")
print(greet("Alice"))
print(add_numbers(5, 3))
print(calculate_average([1.5, 2.5, 3.5]))

# Optional and Union types
def find_user(user_id: int) -> Optional[Dict[str, str]]:
    """Return user data or None if not found"""
    users = {1: {"name": "Alice", "email": "alice@email.com"}}
    return users.get(user_id)

def process_data(data: Union[str, int, float]) -> str:
    """Process different types of data"""
    return f"Processing: {data} (type: {type(data).__name__})"

print("\n=== Optional and Union Types ===")
user = find_user(1)
print(f"User found: {user}")
print(process_data("text"))
print(process_data(42))
print(process_data(3.14))

# Complex type hints
def create_user_mapping(users: List[Tuple[int, str]]) -> Dict[int, str]:
    """Create a mapping from user ID to name"""
    return {user_id: name for user_id, name in users}

users_list: List[Tuple[int, str]] = [(1, "Alice"), (2, "Bob"), (3, "Charlie")]
user_mapping = create_user_mapping(users_list)
print(f"\nUser mapping: {user_mapping}")

# Function type hints
def apply_operation(numbers: List[int], operation: Callable[[int], int]) -> List[int]:
    """Apply an operation to each number"""
    return [operation(num) for num in numbers]

def square(x: int) -> int:
    return x ** 2

numbers = [1, 2, 3, 4, 5]
squared = apply_operation(numbers, square)
print(f"Squared numbers: {squared}")

# Generic types
T = TypeVar('T')

class Stack(Generic[T]):
    def __init__(self) -> None:
        self._items: List[T] = []
    
    def push(self, item: T) -> None:
        self._items.append(item)
    
    def pop(self) -> Optional[T]:
        if self._items:
            return self._items.pop()
        return None
    
    def peek(self) -> Optional[T]:
        if self._items:
            return self._items[-1]
        return None
    
    def is_empty(self) -> bool:
        return len(self._items) == 0

# Using generic stack
print("\n=== Generic Stack ===")
int_stack: Stack[int] = Stack()
int_stack.push(1)
int_stack.push(2)
int_stack.push(3)

print(f"Top item: {int_stack.peek()}")
print(f"Popped: {int_stack.pop()}")
print(f"Is empty: {int_stack.is_empty()}")

string_stack: Stack[str] = Stack()
string_stack.push("hello")
string_stack.push("world")
print(f"String stack top: {string_stack.peek()}")

# Class with type hints
class Person:
    def __init__(self, name: str, age: int, email: Optional[str] = None):
        self.name = name
        self.age = age
        self.email = email
    
    def get_info(self) -> Dict[str, Union[str, int]]:
        info = {"name": self.name, "age": self.age}
        if self.email:
            info["email"] = self.email
        return info
    
    def is_adult(self) -> bool:
        return self.age >= 18

# Using the Person class
person = Person("Alice", 25, "alice@email.com")
print(f"\nPerson info: {person.get_info()}")
print(f"Is adult: {person.is_adult()}")
```

```python
# Custom data structure - Trie
class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_end_word = False

class Trie:
    def __init__(self):
        self.root = TrieNode()
    
    def insert(self, word: str):
        node = self.root
        for char in word:
            if char not in node.children:
                node.children[char] = TrieNode()
            node = node.children[char]
        node.is_end_word = True
    
    def search(self, word: str) -> bool:
        node = self.root
        for char in word:
            if char not in node.children:
                return False
            node = node.children[char]
        return node.is_end_word
    
    def starts_with(self, prefix: str) -> bool:
        node = self.root
        for char in prefix:
            if char not in node.children:
                return False
            node = node.children[char]
        return True
```

## **8. Asyncio & Concurrency**

### Asynchronous Programming
- **Asyncio**: Single-threaded concurrent programming using coroutines
- **async/await**: Keywords for defining and calling coroutines
- **Event Loop**: Manages and executes asynchronous tasks
- **Concurrent Execution**: Multiple tasks running seemingly simultaneously

### Benefits:
- Efficient I/O-bound operations
- Better resource utilization
- Responsive applications

**➤ Asyncio Examples:**


```python
import asyncio
import time
from concurrent.futures import ThreadPoolExecutor, ProcessPoolExecutor
import threading

# Basic async function
async def say_hello(name, delay):
    print(f"Hello {name}, starting...")
    await asyncio.sleep(delay)
    print(f"Hello {name}, finished after {delay}s!")
    return f"Completed: {name}"

# Running async functions
async def main_async_example():
    print("=== Basic Async Example ===")
    start_time = time.time()
    
    # Sequential execution
    print("Sequential execution:")
    await say_hello("Alice", 1)
    await say_hello("Bob", 1)
    
    sequential_time = time.time() - start_time
    print(f"Sequential time: {sequential_time:.2f}s")
    
    # Concurrent execution
    print("\nConcurrent execution:")
    start_time = time.time()
    
    task1 = say_hello("Charlie", 1)
    task2 = say_hello("David", 1)
    
    results = await asyncio.gather(task1, task2)
    concurrent_time = time.time() - start_time
    
    print(f"Concurrent time: {concurrent_time:.2f}s")
    print(f"Results: {results}")

# Run the async example (Note: In Jupyter, you might need to handle event loops differently)
try:
    # For Jupyter notebooks, use this approach

def cpu_bound_task(n):

# Threading for I/O-bound tasks
start_time = time.time()
with ThreadPoolExecutor(max_workers=3) as executor:
    futures = [executor.submit(io_bound_task, 0.5) for _ in range(3)]
    results = [future.result() for future in futures]

# Multiprocessing for CPU-bound tasks

# Async context manager
class AsyncResource:
    def __init__(self, name):

        print(f"Acquiring async resource: {self.name}")

        print(f"Releasing async resource: {self.name}")
        await asyncio.sleep(0.1)  # Simulate cleanup time

async def use_async_resource():
    async with AsyncResource("Database Connection") as resource:
```

## **10. Performance Optimization & Profiling**

### Performance Optimization Techniques
- **Time and Space Complexity**: Choose efficient algorithms
- **Built-in Functions**: Use optimized built-ins when possible
- **List vs Generator**: Use generators for large datasets
- **Caching**: Store expensive computation results
- **Profiling**: Measure performance to identify bottlenecks

**➤ Performance Examples:**


```python
import time

import cProfile
import sys

# Timing decorator
def timing_decorator(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        start_time = time.time()
        result = func(*args, **kwargs)
        end_time = time.time()
        print(f"{func.__name__} took {end_time - start_time:.6f} seconds")
        return result
    return wrapper

# Inefficient vs Efficient approaches
@timing_decorator
def inefficient_sum(n):
    """Inefficient: Create full list in memory"""
    return sum([i**2 for i in range(n)])

@timing_decorator
def efficient_sum(n):
    """Efficient: Use generator expression"""
    return sum(i**2 for i in range(n))

print("=== Efficiency Comparison ===")
n = 100000
result1 = inefficient_sum(n)
result2 = efficient_sum(n)
print(f"Results equal: {result1 == result2}")

    # Generator expression

    print(f"Generator expression memory: {sys.getsizeof(gen_expr)} bytes")
    
    # Set comprehension
    set_comp = {x**2 for x in range(n)}
    print(f"Set comprehension memory: {sys.getsizeof(set_comp)} bytes")

memory_comparison()
```

## 4. Working with *args and **kwargs

- `*args`: Allows variable number of positional arguments.

- `**kwargs`: Allows variable number of keyword arguments.

**➤ Example with \*args:**


```python
def add(*numbers):
    total = sum(numbers)
    print("Sum:", total)

add(1, 2, 3)
add(4, 5)
```

**➤ Example with \*\*kwargs:**


```python
def student_info(**kwargs):
    for key, value in kwargs.items():
        print(f"{key}: {value}")

student_info(name="Alice", age=21, grade="A")
```

```python
nums = [1, 2, 3, 4]
```

```python
nums = [1, 2, 3, 4, 5, 6]
```

- `reduce()` applies a function cumulatively to the items of a sequence.

- It is part of the `functools` module.


```python
from functools import reduce

nums = [1, 2, 3, 4]

product = reduce(lambda x, y: x * y, nums)
```
